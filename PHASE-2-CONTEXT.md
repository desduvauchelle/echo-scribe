# Phase 2: AI Processing Pipeline with MLX

## Goal
Replace the stub `AIProcessor` with real local LLM inference using Apple MLX. After a user speaks, the AI should analyze the transcript and extract: cleaned text, summary, tasks with due dates, project assignment, and tags.

## Current State (Phase 1 complete)
- App builds and runs via `EchoScribe.xcodeproj` (generated by xcodegen from `project.yml`)
- Speech-to-text works via `AppleSpeechService` using `SFSpeechRecognizer`
- Notes are saved to SQLite via GRDB
- `AIProcessor.swift` is a **stub** that returns raw text as-is with no enrichment
- The full pipeline exists: transcription → AIProcessor → NoteProcessingPipeline → DB

## Tech Stack
- Swift 6.0 / macOS 15+ / SwiftUI
- GRDB.swift 7.10 for SQLite
- Xcode project via xcodegen (`project.yml`)
- All ViewModels and services are `@MainActor`
